# 文档使用指南

## 📚 文档体系概览

本项目包含完整的文档体系，帮助AI和开发者高效协作。以下是各文档的用途和使用顺序。

---

## 🎯 快速开始

### 第一次接触项目？

1. **阅读** `CLAUDE.md` - 了解项目整体情况
2. **阅读** `PROGRESS.md` - 查看当前进度
3. **阅读** `MIGRATION_PLAN.md` - 了解整体移植方案
4. **开始执行任务** - 参考 `DETAILED_TASK_GUIDE.md`

### 准备执行任务？

1. **查看任务列表** - `MIGRATION_PLAN.md` 或 `MIGRATION_PLAN_SMALL_STEPS.md`
2. **查看详细步骤** - `DETAILED_TASK_GUIDE.md`（超详细，包含完整代码）
3. **执行任务** - 按照详细步骤操作
4. **更新进度** - 完成后更新 `PROGRESS.md`

## 🎯 使用场景

### 场景1: 我要开始一个新任务

**步骤**：
1. 打开 `PROGRESS.md`，查看"下一步任务"
2. 打开 `DETAILED_TASK_GUIDE.md`，找到对应任务的详细步骤
3. 按照详细步骤执行：
   - ✅ 前置条件检查
   - ✅ 查看Unity源码参考
   - ✅ 执行详细步骤（每个步骤都有具体操作）
   - ✅ 复制完整代码示例
   - ✅ 验证结果
   - ✅ 处理常见问题

**示例**：
```
任务：1.1.1 创建配置管理器基类

1. 打开 DETAILED_TASK_GUIDE.md
2. 找到"任务 1.1.1"
3. 按照步骤执行：
   - 步骤 1.1.1.1: 创建目录结构
   - 步骤 1.1.1.2: 创建 ConfigManager.ts 文件（复制完整代码）
   - 步骤 1.1.1.3: 验证代码编译
4. 完成后更新 PROGRESS.md
```

### 场景2: 我需要了解代码转换规范

**步骤**：
1. 打开 `CODE_CONVERSION_GUIDE.md`
2. 查找对应的转换场景（如：MonoBehaviour → Component）
3. 参考代码示例进行转换

### 场景3: 我遇到了问题

**步骤**：
1. 查看 `DETAILED_TASK_GUIDE.md` 中的"常见问题"部分
2. 如果问题未解决，查看 `ISSUES.md`
3. 如果仍未解决，在 `ISSUES.md` 中记录新问题

### 场景4: 我需要转换资源

**步骤**：
1. 打开 `ASSET_CONVERSION_GUIDE.md`
2. 查看对应的资源类型转换规范
3. 使用 `TOOLS_GUIDE.md` 中的工具进行批量转换

## 🔄 文档更新流程

### 完成任务后

1. **更新 `PROGRESS.md`**
   - 标记任务为 ✅ 已完成
   - 记录完成日期
   - 更新"下一步任务"

2. **更新 `DETAILED_TASK_GUIDE.md`**（如需要）
   - 添加新任务的详细步骤
   - 记录遇到的问题和解决方案

3. **更新 `ISSUES.md`**（如遇到问题）
   - 记录问题
   - 记录解决方案

### 发现文档问题

1. 在对应文档中直接修改
2. 更新文档的"最后更新"日期
3. 如果问题影响其他文档，同步更新
